## âš™ï¸ Configuration

### 1. Database Configuration

Edit `.env` file and configure your database:

**For MySQL:**

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=ipharma_mart
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

**For SQLite (Development):**

```env
DB_CONNECTION=sqlite
DB_DATABASE=/absolute/path/to/database.sqlite
```

### 2. Application Configuration

```env
APP_NAME="iPharma Mart"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

# Session & Cache
SESSION_DRIVER=file
CACHE_DRIVER=file
QUEUE_CONNECTION=sync
```

## ğŸ—„ Database Setup

### 1. Create Database

**MySQL:**

```bash
mysql -u root -p
CREATE DATABASE ipharma_mart;
exit
```

**SQLite:**

```bash
touch database/database.sqlite
```

### 2. Run Migrations

```bash
php artisan migrate
```

### 3. Seed Database (Recommended)

Populate with sample data including Philippine-localized information:

```bash
php artisan db:seed
```

This creates:

-   1 Admin user
-   10 Categories (Medicines, Vitamins, First Aid, etc.)
-   5 Suppliers (with Davao City addresses)
-   10 Customers (with Philippine phone numbers)
-   50 Products (with realistic pharmacy items)
-   20 Sample sales transactions

## ğŸ¯ Key Features Explained

### Point of Sale (POS) System

The POS is optimized for cashier operations:

1. **Fullscreen Requirement**: Opens with a modal requiring fullscreen mode (F11) for focus
2. **Product Selection**: Grid layout with search and category filters
3. **Shopping Cart**: Table format showing item details, quantity controls, and prices
4. **Customer Management**:
    - Optional customer selection
    - Quick "Add Customer" modal for new customers
    - Supports walk-in customers (default)
5. **Checkout**: Creates sale record and generates receipt

### Inventory Management

-   **Low Stock Alerts**: Visual indicators when stock falls below threshold
-   **Category Organization**: Products grouped by categories
-   **Stock Tracking**: Real-time updates when sales are made
-   **CRUD Operations**: Full create, read, update, delete functionality

### Design System

Custom CSS variables provide consistent theming:

-   **Primary Green**: #2C6356 (Dark) / #3A7D6F (Light)
-   **Accent Orange**: #FF9052
-   **Typography**: Inter font family
-   **Responsive Breakpoints**: Mobile (<640px), Tablet (640-1024px), Desktop (>1024px)

## ğŸ› Troubleshooting

### Common Issues

**1. Page Expired Error (CSRF Token)**

```bash
# Clear cache and restart
php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan view:clear
php artisan serve
```

**2. Assets Not Loading**

```bash
# Rebuild assets
npm run build
```

**3. Database Connection Error**

-   Check `.env` database credentials
-   Ensure database exists
-   Verify database service is running

**4. Permission Errors (Linux/Mac)**

```bash
# Fix storage and cache permissions
chmod -R 775 storage bootstrap/cache
```

**5. Composer Dependencies Issue**

```bash
# Update dependencies
composer update
composer dump-autoload
```
